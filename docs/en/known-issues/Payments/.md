## Summary

When a **change order to a higher value** is processed using the **BrasPag** connector, the new transaction generated is not captured at the same speed as a regular transaction — causing a delay in the order flow. Additionally, even if the new transaction is captured via **auto-capture**, the order remains stuck and requires **manual intervention** to proceed.

## Simulation

Unable to reproduce in a controlled environment. The issue manifests in production on marketplace orders using BrasPag where a change order to a higher value has been applied.

## Workaround

Yes, there is a workaround available: manually trigger the capture of the transaction using the **capture API**, which allows the order flow to continue normally after the manual capture is executed.
